new (opts)

Create a new configuration storage client instance.

Parameters:

  • opts (table)

    Client options.

    May contain driver specific options (listed in conf.client.etcd.new).
    • driver (string)

      Driver name. Only 'etcd' is supported now.

      It is the mandatory option.
    • endpoints ({string, ...})

      Endpoint URLs.

Returns:

    client instance.

Raises:

See 'General API notes' in the etcd client documentation.

XXX: Move the General API notes (at least errors) outside of the etcd client documentation.

Usage:

    local conf_lib = require('conf')

local urls = {
    'http://localhost:2379',
    'http://localhost:2381',
    'http://localhost:2383',
}
local conf = conf.new({driver = 'etcd', endpoints = urls})

Instance methods

instance.get (self, key)

Fetch a value.

Note: A string value that can be interpreted as a number will be converted to a number. It is constraint of current implementation and will be resolved in a future.

Parameters:

  • self

    Client instance.
  • key (string)

    A key to fetch.

    A dot notation may be used to access a nested map field or array element: 'foo.bar' or 'foo.1'. Array indices are 1-based, just as in Lua.

    XXX: Describe the dot notition is the separate section (like 'General API notes').

Returns:

    Response of the following structure: 

     {
     data = obj,
 }

    obj is the value associated with given key or nil if given key does not exist.

Raises:

See 'General API notes' in the etcd client documentation.

XXX: Move the General API notes (at least errors) outside of the etcd client documentation.

Usage:

    -- Put a map.
conf:set('foo', {bar = {baz = 42}})

-- Fetch it back.
local res = conf:get('foo')
-- res.data is {bar = {baz = 42}}

-- Or fetch a field.
local res = conf:get('foo.bar')
-- res.data is {baz = 42}
local res = conf:get('foo.bar.baz')
-- res.data is 42

-- Put an array.
conf:set('foo', {'a', 'b', 'c'})

-- Fetch an array element.
local res = conf:get('foo.1')
-- res.data is 'a'
instance.set (self, key, obj)

Store a value.

If given key contains a value, the value is **replaced** with the new one. Maps are NOT merged.

Use dot notation to set a particular field of a map (or particular element of an array).

Storing of foo.bar.baz means that foo and foo.bar will store a map. The same for foo.1: foo will store an array.

XXX: Define behaviour for corner cases like storing foo.bar, when foo exists and it is not a map.

Parameters:

  • self

    Client instance.
  • key (string)

    A key to set.

    A dot notation may be used, see conf.get.
  • obj

    A value to store.

    Allowed types: string, number, table (a map or an array).

Returns:

    Nothing.

    It is recommended to don't lean on the fact that the method does not return anything, because the API can be extended with a return value in a future.

Raises:

See 'General API notes' in the etcd client documentation.

XXX: Move the General API notes (at least errors) outside of the etcd client documentation.

Usage:

    -- Store a scalar, a map and an array.
conf:set('foo', 'hello')
conf:set('moo', {bar = {baz = 42}})
conf:set('xoo', {'a', 'b', 'c'})

-- Set a nested field and an array element.
conf:set('foo.bar.baz', 'hello')
conf:set('foo.1', 'hello')
